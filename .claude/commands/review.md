# /review — 静的コードレビュー

Evaluator の静的レビューモードを起動し、コード差分または指定ファイルをレビューする。

## 手順

1. **レビュー対象の決定**

   引数の有無に応じてレビュー対象を決定する：

   - **引数なし**: `git diff HEAD --name-only` を実行し、変更ファイル一覧を取得する
   - **ファイル指定（例: `$ARGUMENTS` が単一ファイルパス）**: そのファイルをレビュー対象にする
   - **ディレクトリ/パス指定（例: `$ARGUMENTS` がディレクトリ）**: 指定パス配下のファイルを再帰的に列挙する

2. **対象拡張子でフィルタリング**

   以下の拡張子のファイルのみを対象とする。それ以外は除外する：
   - `.ts` `.tsx` `.css` `.json` `.mjs` `.js`

3. **対象ファイル 0 件チェック**

   フィルタリング後に対象ファイルが 0 件の場合、以下のメッセージを表示して終了する：

   > レビュー対象のファイルが見つかりませんでした。対象拡張子: .ts .tsx .css .json .mjs .js

4. **Evaluator 静的レビューモードでレビュー実行**

   `.claude/agents/evaluator.md` の「静的レビューモード」に従い、以下の観点でレビューを行う：

   - **コード品質** — 型安全性、コンポーネント設計、可読性、重複コード
   - **プロジェクト規約遵守** — CLAUDE.md のデザインシステム・命名規則・既存コードベースの規約
   - **セキュリティ・アクセシビリティ** — XSS リスク、ARIA 属性、セマンティック HTML
   - **パフォーマンス** — 不要な再レンダリング、バンドルサイズ、画像最適化

   各指摘には severity（**critical** / **warning** / **info**）を付与する。

   過剰指摘の抑止ガイドラインに従う：
   - 1ファイルあたり最大 8 件を目安。超える場合は severity が高い指摘を優先
   - 同一パターンの問題は代表 1 箇所を指摘し「他 N 箇所にも同様の問題あり」と記載
   - 些末な指摘（スタイルの好み、動作に影響しない微細な差異）は行わない

5. **結果をチャットに表示**

   `.claude/agents/evaluator.md` の「出力フォーマット」に従い、以下の形式でチャットに直接表示する：

   - **ファイルごとの指摘リスト**: ファイルパス・行番号（特定できる場合）・severity・カテゴリ・指摘内容
   - **指摘 0 件のファイル**: 「問題なし」と簡潔に表示
   - **全体サマリー**: 対象ファイル数、severity 別件数

   ファイル保存は行わない。合格/不合格の判定は行わない（参考情報のみ）。
