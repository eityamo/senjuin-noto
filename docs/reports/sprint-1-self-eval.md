# スプリント 1 自己評価レポート

## 実装した機能

- **機能 1: Evaluator への静的コードレビューモード追加**: `.claude/agents/evaluator.md` を2モード制（ブラウザ検証モード / 静的レビューモード）に再構成した
- **機能 2: 起動インターフェースとレビュー対象の決定**: `.claude/commands/review.md` を新規作成し、`/review` スラッシュコマンドを定義した

## 受け入れ基準の達成状況

### 機能 1

- [x] `.claude/agents/evaluator.md` に「ブラウザ検証モード」と「静的レビューモード」の 2 モードが明記されている
- [x] 各モードの役割・呼び出し条件・判定方式の違いが明確に記述されている（冒頭のモード概要テーブルで比較）
- [x] 静的レビューモードのレビュー観点が 4 つ（コード品質 / プロジェクト規約遵守 / セキュリティ・アクセシビリティ / パフォーマンス）に整理されている
- [x] severity が critical / warning / info の 3 段階で定義されている
- [x] レビュアー像として「シニアエンジニア。過剰指摘を避け、本当に修正価値のある指摘のみを行う」ことが明記されている
- [x] 対象拡張子（`.ts` `.tsx` `.css` `.json` `.mjs` `.js`）がルールとして明示されている
- [x] 静的レビューモードではブラウザ操作・UI 検証を行わないことが明記されている
- [x] ブラウザ検証モードの既存内容（Playwright MCP による検証、評価基準と閾値、レポート出力先など）が損なわれていない
- [x] 既存コードベース（Next.js 16 / React 19 / Tailwind CSS 4 / TypeScript strict）の規約に言及している

### 機能 2

- [x] スラッシュコマンド `/review` から静的レビューモードが起動できる（`.claude/commands/review.md` を作成）
- [x] 自然言語（日本語の「レビューして」「コードレビューお願い」等）から静的レビューモードが起動できる（evaluator.md の「呼び出し方」セクションに記載）
- [x] どちらの入り口から起動しても、同一のレビュー観点・severity 区分・出力フォーマットが適用される
- [x] 引数なしで呼び出した場合、`git diff HEAD` の変更ファイルのみがレビュー対象になる
- [x] 引数で単一ファイルを指定した場合、そのファイルがレビュー対象になる
- [x] 引数でディレクトリ／glob 風のパスを指定した場合、配下の対象拡張子ファイルがレビュー対象になる
- [x] 対象拡張子以外のファイルは、対象ファイル選定の段階で自動的に除外される
- [x] 対象ファイルが 0 件の場合、その旨をチャットに明示して終了する
- [x] 呼び出し方の例が Evaluator 定義に明記されている（evaluator.md の「呼び出し方」セクション）
